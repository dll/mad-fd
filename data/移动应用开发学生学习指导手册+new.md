# 移动应用开发学生学习指导手册

## 📚 课程概览

### 🎯 课程目标
本课程旨在培养学生掌握移动应用开发的多元技术体系（原生/混合/小程序/多端），理解不同开发模式的适用场景，熟悉主流跨平台开发框架及 AI 编程工具的使用，具备跨平台应用系统分析，设计和开发能力，能够运用 RESTful API 实现移动端与后端的数据交互，为从事移动开发工作及毕业设计奠定基础。

### 📖 课程结构
- **理论学习**：6 章系统性理论知识（24 学时）
- **实验实践**：6 个实验项目（24 学时）
- **考核方式**：平时（20%）+ 实验（30%）+ 期末大作业（50%）

### 📋 先修课程
本课程要求已修读：面向对象程序设计、数据库原理，软件工程。后端交互、数据库操作、工程规范等内容将在已有基础上展开。

### 📊 课时分配明细
| 教学模块 | 学时分配 | 具体安排 | 学习重点 |
|----------|----------|----------|----------|
| **理论教学** | **24 学时** | 每章 4 学时 × 6 章 | 技术体系、核心概念、框架对比 |
| **实验教学** | **24 学时** | 实验 1（2h）+ 实验 2-5（各 4h）+ 实验 6（6h） | AI 新范式开发、技术栈探究、团队协作 |
| **总计课时** | **48 学时** | 理论 24 + 实验 24 | 理论实践并重，全面培养能力 |

### 🏆 学习成果
通过本课程学习，您将获得：
- ✅ 7 大主流移动开发技术栈（Android/iOS/Flutter/React Native/Uniapp/MAUI/鸿蒙）的实践经验
- ✅ AI 新范式开发流程（需求→编码→测试→运维）的完整体验
- ✅ RESTful API 后端交互与移动硬件传感器调用能力
- ✅ Git 版本控制与团队协作经验
- ✅ 面向就业的核心竞争力

## 📖 理论学习指导

### 第一章：移动应用开发技术体系（4 学时）
**学习重点**：移动应用分类、技术选型方法论，开发工具链

**核心概念**：原生应用、混合应用、小程序、多端应用、跨平台技术路线

**重要知识点**：
- 原生应用 vs 混合应用 vs 小程序 vs 多端应用的技术特征与适用场景
- Android/iOS 生态、微信小程序平台、华为鸿蒙多端框架对比
- Android Studio、Xcode、HBuilderX、DevEco Studio、Visual Studio（MAUI）工具链
- TRAE 等 AI 编程工具在移动开发中的辅助作用
- 技术选型决策模型：基于性能/成本/跨平台性/团队技术栈

**学习建议**：
- 📱 理解原生（Flutter/MAUI）、JS 桥接（React Native）、Web 封装（Uniapp）三条技术路线的差异
- 📊 掌握基于性能/成本/跨平台性/团队技术栈的技术选型决策模型
- 🔧 熟悉 Android Studio、Xcode、HBuilderX、DevEco Studio、Visual Studio（MAUI）等工具链
- 🤖 了解 TRAE 等 AI 编程工具在移动开发中的辅助作用

### 第二章：原生开发基础（4 学时）
**学习重点**：Android Kotlin 开发、iOS SwiftUI 概述、原生与跨平台对比

**核心技术**：Kotlin、Activity 生命周期、SwiftUI、Logcat 调试

**重要知识点**：
- Kotlin 语言基础（空安全、Lambda、协程）
- Android 四大组件与 Activity 生命周期（onCreate/onStart/onResume/onPause/onStop/onDestroy）
- UI 控件：TextView、Button、EditText、RecyclerView
- Intent 与 Activity 跳转、数据传递（Bundle/Parcelable）
- iOS ViewController 生命周期（viewDidLoad/viewWillAppear/viewDidAppear）
- SwiftUI 声明式编程基础
- 原生开发 vs 跨平台开发的性能与成本对比

**学习建议**：
- 🔧 重点掌握 Kotlin 语言基础与 Android 核心组件
- 📝 理解 Activity 生命周期管理与 UI 控件使用
- 🍎 iOS 部分以理论理解为主，掌握 ViewController 架构与 SwiftUI 组件模型
- ⚖️ 思考原生开发的性能优势与开发成本

### 第三章：跨平台应用开发（4 学时）
**学习重点**：Flutter/React Native/Uniapp/MAUI 框架，后端交互、移动硬件调用

**核心概念**：跨平台框架对比、RESTful API、JSON 解析、Token 认证、传感器接口

**重要知识点**：
- Flutter：Dart 语言基础、Widget 组件体系、状态管理（Provider/Bloc）
- React Native：JSX 语法、原生模块桥接、组件生命周期、Hooks
- Uniapp：Vue 语法基础、多端编译原理、条件编译
- MAUI：C# 跨平台 UI 组件、MAUI.Android/MAUI.iOS 平台特性
- RESTful API 设计规范与 HTTP 请求库（Dio/Axios/HttpClient）
- JSON 数据解析与 Token 认证机制
- 移动硬件能力：GPS 定位、加速度计、陀螺仪等传感器接口

**学习建议**：
- 🌉 对比理解四大跨平台框架的技术特点与适用场景
- 🔗 重点掌握 RESTful API 交互流程
- 📱 了解移动设备传感器调用与物联网数据采集
- 🤖 学会使用 TRAE 辅助生成基础代码与调试

### 第四章：微信小程序开发（4 学时）
**学习重点**：MINA 框架、页面路由与数据绑定，云开发基础

**核心概念**：WXML/WXSS 语法、小程序生命周期、云函数、云数据库

**重要知识点**：
- 微信小程序 MINA 框架架构（双线程模型：逻辑层+渲染层）
- WXML 标记语言：数据绑定、列表渲染（wx:for）、条件渲染（wx:if）
- WXSS 样式语言：选择器、flex 布局、rpx 响应式单位
- 小程序生命周期（onLoad/onShow/onReady/onHide/onUnload）
- 页面路由与数据传递（URL 参数、globalData）
- 小程序云开发：云函数、云数据库、云存储
- Taro 框架实现多端小程序代码复用

**学习建议**：
- 📱 掌握微信小程序页面路由、数据绑定与生命周期
- ☁️ 了解小程序云开发（云函数，云数据库）基础
- 🔄 了解 Taro 框架实现多端小程序代码复用（拓展）
- 🤖 使用 TRAE 辅助生成小程序页面代码

### 第五章：鸿蒙多端应用开发（4 学时）【必须技术栈】
**学习重点**：HarmonyOS NEXT 架构、ArkUI/ArkTS、多端适配、传感器与物联网、分布式能力

**核心概念**：声明式 UI、状态管理、分布式软总线、跨设备数据同步

**重要知识点**：
- HarmonyOS NEXT 架构与 Android/iOS 生态对比
- DevEco Studio 开发环境配置
- ArkTS 语言基础：类型系统、接口、泛型
- ArkUI 声明式 UI 框架：@Component、@State、@Prop、@Link 装饰器
- 基础组件：Text、Button、Column、Row、Stack、Grid
- 多端部署：手机/平板界面自适应布局策略（断点系统、响应式设计）
- 分布式能力：分布式软总线、分布式数据管理、分布式任务调度
- 传感器数据采集：光线传感器、陀螺仪、加速度计
- 物联网应用场景：智能家居、智慧办公、车联网

**学习建议**：
- 🔧 掌握 DevEco Studio 开发环境与 ArkTS/ArkUI 声明式 UI 语法
- 📱 理解手机/平板界面自适应布局策略
- 🌐 理解分布式能力原理（通过模拟器演示）
- 📡 了解传感器数据采集与物联网应用场景
- ⚠️ **鸿蒙是本课程的必须技术栈**，所有同学都需要掌握

### 第六章：综合开发实践（4 学时）
**学习重点**：架构设计，数据存储，性能优化、Git 协作、AI 工具深度应用

**核心概念**：MVP/MVVM 模式、Git 分支管理、代码审查、AI 辅助重构

**重要知识点**：
- MVVM 架构模式：Model-View-ViewModel 分离
- 数据存储方案：SharedPreferences、SQLite、小程序本地缓存、鸿蒙数据管理
- 性能优化：启动速度优化、UI 渲染效率提升、网络请求优化
- Git 版本控制：分支管理（main/dev/feature）、代码合并、冲突解决
- 代码审查：Pull Request、Code Review 流程
- AI 工具深度应用：代码重构、性能优化建议、自动化测试生成

**学习建议**：
- 🏗️ 理解 MVP/MVVM 架构模式在移动应用中的实践
- 💾 掌握多种数据存储方案
- ⚡ 学习启动速度优化、UI 渲染效率提升，网络请求优化
- 🔀 掌握 Git 分支管理、代码审查、团队协作开发流程
- 🤖 学会利用 TRAE 进行代码重构与性能优化分析

## 🔬 实验学习指导

> **AI 新范式开发流程**：所有实验均采用模板化四阶段流程——**① 需求分析 → ② AI 辅助编码 → ③ 测试验证 → ④ 部署运维**。

> **分组模式**：每组 6 人，按两个维度分组：
> - **项目业务组**（3-4组）：每组选择一个业务项目方向
> - **开发技术组**（6人/组）：每人负责一个技术栈，独立完成后组内技术分享

### 实验项目维度说明

#### 项目业务组（选题方向）

| 智慧健康 | 运动健康 | 步数统计、心率监测、运动记录 |

#### 开发技术组（技术分工）
每组 6 人，每人负责一个技术栈：
| 序号 | 技术栈 | 开发语言 | 开发工具 |
|------|--------|----------|----------|

**实验目标**：搭建各平台开发环境，成功运行 Hello World 项目。

**技术栈分工**：每人负责一个平台（Android Studio / Xcode / Flutter / React Native / HBuilderX / DevEco Studio）。

**AI 新范式流程**：
1. **需求分析**：明确各平台环境依赖与版本要求
2. **AI 辅助编码**：使用 TRAE 生成 Hello World 模板代码
3. **测试验证**：在模拟器/真机上验证项目运行
4. **部署运维**：记录环境配置文档，总结常见问题与解决方案

**提交要求**：环境配置截图 + 运行成功截图 + 配置问题记录文档。

## 📝 学习方法与策略

### 🤖 AI 工具使用指南（TRAE）
TRAE 是本课程推荐的 AI 编程辅助工具，贯穿所有实验环节：
- **代码生成**：输入需求描述，TRAE 生成基础代码框架
- **调试辅助**：粘贴错误信息，获取修复建议
- **代码重构**：选中代码段，请求优化建议
- **学习辅助**：询问技术概念，获取示例代码

**使用建议**：
- AI 生成的代码必须理解后再使用，不要盲目复制
- 将 AI 作为学习加速器，而非替代思考的工具
- 记录 AI 辅助开发的过程，作为实验报告的一部分

### 🔀 Git 协作指南
实验六要求使用 Git 进行团队协作：
- **基本流程**：clone → branch → commit → push → pull request → merge
- **分支策略**：main（主分支）+ dev（开发分支）+ feature/xxx（功能分支）
- **提交规范**：使用清晰的 commit message，如 `feat: 添加登录页面` `fix: 修复列表刷新bug`
- **代码审查**：组内互相 review 代码，提出改进建议

## ❓ 常见问题 FAQ

**Q1: 没有 Mac 电脑，iOS 实验怎么做？**
A: iOS 部分以理论讲授与教师演示为主，实验中可选择其他技术栈（Android/Flutter/React Native/Uniapp/MAUI/鸿蒙）独立完成。

**Q2: 6 个技术栈都要学会吗？**
A: 不需要。每人深入掌握 1 个技术栈，通过组内分享了解其他技术栈的特点与差异，重点培养技术对比分析能力。**注意：鸿蒙是必须技术栈，所有同学都需要掌握**。

**Q3: AI 新范式开发流程具体怎么操作？**
A: 每个实验按四阶段执行：① 先写需求文档 → ② 用 TRAE 辅助编码 → ③ 编写测试用例验证 → ④ 打包部署并撰写总结。实验报告中需体现每个阶段的过程记录。

**Q4: TRAE 工具怎么获取？**
A: 访问 https://trae.ai 下载安装，课堂上教师会演示基本使用方法。

**Q5: Git 不熟悉怎么办？**
A: 第六章理论课会讲解 Git 基础，实验六前会有 Git 操作演练。建议提前学习 Git 基本命令（clone/add/commit/push/pull/branch/merge）。

**Q6: 后端 API 从哪里来？**
A: 实验三中的 RESTful API 可使用公开的免费 API 服务（如 JSONPlaceholder），也可由教师提供统一的后端接口。学生已修数据库原理和软件工程，具备理解 API 交互的基础。

**Q7: 期末大作业是个人还是团队？**
A: 期末大作业采用团队形式，延续实验六的分组模式，每人负责一个技术栈，提交多端应用 + 技术选型报告 + 项目文档。

**Q8: 移动开发就业前景如何？**
A: 跨平台开发（Flutter/React Native/Uniapp）需求持续增长，微信小程序市场需求量大，**鸿蒙开发是国产化趋势国家重点**，是就业核心竞争力。

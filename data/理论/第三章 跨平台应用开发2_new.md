学前测验：后端交互与硬件调用基础认知

### 测验说明
本测验检测学习者对RESTful API、HTTP请求、移动硬件调用等核心概念的认知，帮助确定本章进阶学习起点。

### 第1题
**题目**：RESTful API的核心设计原则是什么？

A. 使用XML格式传输数据
B. 基于HTTP协议的资源导向架构
C. 必须使用 SOAP 协议
D. 数据存储在客户端

**正确答案**：B

**解释**：RESTful API基于HTTP协议，采用资源导向的架构风格，通过URL定位资源，使用HTTP方法（GET/POST/PUT/DELETE）操作资源，是现代移动应用后端交互的主流方案。

### 第2题
**题目**：以下哪个HTTP方法通常用于获取数据？

A. POST
B. PUT
C. GET
D. DELETE

**正确答案**：C

**解释**：GET方法用于从服务器获取数据，是最常用的读取数据的HTTP方法。POST用于创建数据，PUT用于更新数据，DELETE用于删除数据。

### 第3题
**题目**：Token认证机制的主要作用是什么？

A. 加密传输数据
B. 用户身份验证与会话管理
C. 提高网络请求速度
D. 压缩数据体积

**正确答案**：B

**解释**：Token认证是一种无状态的身份验证机制，用户登录成功后服务器返回Token，客户端在后续请求中携带Token来验证身份，无需在服务器端存储会话信息。

### 第4题
**题目**：移动应用中获取位置信息通常使用哪个传感器或API？

A. 加速度计
B. GPS/Location API
C. 陀螺仪
D. 摄像头

**正确答案**：B

**解释**：获取位置信息通常通过GPS（全球定位系统）或网络定位API实现。加速度计用于检测设备运动状态，陀螺仪用于检测设备旋转角度，摄像头用于拍照和视频。

### 第5题
**题目**：AI编程工具在开发中的主要辅助作用是？

A. 自动部署应用
B. 代码自动生成与智能补全
C. 服务器运维
D. 数据库管理

**正确答案**：B

**解释**：AI编程工具如TRAE的主要作用是根据开发者的注释和上下文自动生成代码、提供智能补全建议，帮助提高开发效率。

### 幻灯片2：RESTful API设计规范
- **标题**：RESTful API设计与实践
- **内容**：
  - **核心原则**：
    - 资源导向：使用名词表示资源
    - HTTP方法对应操作：GET/POST/PUT/DELETE
    - 状态码语义化：200/400/401/404/500
  - **URL设计规范**：
    ```
    GET    /api/users          # 获取用户列表
    GET    /api/users/123      # 获取单个用户
    POST   /api/users          # 创建用户
    PUT    /api/users/123      # 更新用户
    DELETE /api/users/123      # 删除用户
    ```
  - **最佳实践**：
    - 版本控制：/api/v1/
    - 过滤排序：?status=active&sort=name
    - 分页支持：?page=1&size=10

- **内容**：


  - **React Native：Axios**：
    ```javascript
    const response = await axios.get('/api/users');
    const users = response.data.data;
    ```
  - **Uniapp：uni.request**：
    ```javascript
    uni.request({
      url: '/api/users',

    });
    ```

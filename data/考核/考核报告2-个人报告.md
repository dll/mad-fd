- 必须清晰展示对象间的交互时序
- 必须包含参与者、对象、消息、生命线

**顺序图必须包含的元素**：
- 参与者（Actor）：用户或外部系统
- 对象（Object）：参与交互的类实例
- 生命线（Lifeline）：对象存在的时间线
- 消息（Message）：对象间的调用
- 激活框（Activation）：方法执行期间

**顺序图示例结构**：
```
参与者       对象A       对象B       对象C
  |           |           |           |
  |--消息1--> |           |           |
  |           |--消息2--> |           |
  |           |           |--消息3--> |
  |           |           |<--返回3-- |
  |           |<--返回2-- |           |
  |<--返回1-- |           |           |
```

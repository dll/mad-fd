# 移动应用开发教师教学指导手册

## 📚 课程教学概览

### 🎯 课程定位与目标

#### 课程性质
本课程是计算机科学与技术专业的限选课程（选修），开设于第六学期。先修课程为面向对象程序设计、数据库原理、软件工程。总学时 48（理论 24 + 实验 24），2.5 学分，考查课。

#### 教学目标

**知识目标**：
- 理解移动应用开发技术体系（原生/混合/跨平台）和技术选型方法论
- 掌握 Android（Kotlin）原生开发核心技术，了解 iOS（SwiftUI）开发架构
- 掌握跨平台框架（Flutter/React Native/Uniapp/MAUI）的基本原理与开发方法
- 掌握微信小程序开发与**鸿蒙多端应用开发技术**（必须技术栈）
- 理解 RESTful API 交互、移动硬件传感器调用与物联网基础

**能力目标**：
- 具备跨平台应用系统分析、设计和开发能力
- 能够运用 RESTful API 实现移动端与后端数据交互
- 具备技术方案调研、对比分析与选型评估能力
- 掌握 AI 编程工具（TRAE）辅助开发的能力
- 具备 Git 版本控制与团队协作能力

**素质目标**：
- 培养工程思维和创新意识
- 增强问题分析和解决能力
- 提升团队协作和沟通能力
- 培养科技自立自强意识与技术判断力
- **重点培养鸿蒙国产化技术能力**

#### 课程目标与毕业要求映射

| 课程目标 | 权重 | 支撑毕业要求 | 考核重点 |

## 📖 理论教学指导

### 第一章：移动应用开发技术体系（4 学时）

#### 🎯 教学目标
- 让学生了解移动应用开发的技术分类（原生/混合/小程序/多端）
- 理解不同技术路线的优劣与适用场景
- 掌握主流开发工具链的基本使用
- 了解 TRAE 等 AI 编程工具的辅助作用

#### 📝 教学设计

**第 1-2 学时：技术体系概览**
- 导入：展示国内外主流 App 的技术栈选择案例
- 讲授：原生应用、混合应用、小程序、多端应用的技术特征
- 对比：Android/iOS 生态、微信小程序、**华为鸿蒙框架**
- 讨论：为什么不同企业选择不同的技术路线？

**第 3-4 学时：工具链与技术选型**
- 演示：Android Studio、HBuilderX、DevEco Studio、Visual Studio（MAUI）等工具
- 讲授：跨平台技术路线对比——Flutter/MAUI（原生共享）vs React Native（JS 桥接）vs Uniapp（Web 封装）
- 演示：TRAE AI 编程工具的基本使用
- 案例：基于项目需求（性能/成本/跨平台性/团队技术栈）的技术选型决策

#### 💡 教学建议
- 使用真实企业案例（如微信、抖音、支付宝的技术选型）引发学生兴趣
- 工具链演示建议现场操作，让学生直观感受不同 IDE 的特点
- 技术选型讨论可设计为小组辩论形式，培养批判性思维
- **强调鸿蒙生态的国产化意义和发展前景**

#### 🏛️ 课程思政融入
- 通过国内外移动生态竞争案例（华为鸿蒙 vs Android/iOS），培养科技自立自强意识
- 讨论技术选型中的伦理考量（用户隐私、数据安全）

### 第二章：原生开发基础（4 学时）

#### 🎯 教学目标
- 掌握 Kotlin 语言基础与 Android 核心组件
- 理解 iOS 开发基本架构（SwiftUI，理论为主）
- 理解原生开发与跨平台开发的性能与成本差异

#### 📝 教学设计

**第 1-2 学时：Android Kotlin 开发**
- 讲授：Kotlin 语言基础（与 Java 对比，突出简洁性）
- 讲授：Activity 生命周期、UI 控件（TextView/Button/RecyclerView）
- 演示：Logcat 调试工具使用
- 实操引导：简单页面交互示例

**第 3-4 学时：iOS 概述与原生对比**
- 讲授：ViewController 架构、SwiftUI 声明式 UI 组件模型
- 演示：Xcode 开发环境与 SwiftUI 预览（教师演示，无需学生操作）
- 对比分析：原生开发的性能优势与开发成本
- 讨论：什么场景下必须选择原生开发？

#### 💡 教学建议
- Kotlin 部分建议与 Java 对比讲解，学生已有面向对象基础
- iOS 部分以教师演示为主，实验室无 Mac 设备，不要求学生实操
- 可准备 Android vs iOS 开发体验对比的短视频辅助教学

#### 🏛️ 课程思政融入
- 从系统底层逻辑培养严谨的工程思维与问题溯源能力

### 第三章：跨平台应用开发（4 学时）

#### 🎯 教学目标
- 掌握 Flutter/React Native/Uniapp/MAUI 四大框架的基本原理
- 掌握 RESTful API 交互流程（JSON 解析、Token 认证）
- 了解移动设备传感器接口调用与物联网数据采集基础
- 学会使用 TRAE 辅助跨平台开发

#### 📝 教学设计

**第 1-2 学时：跨平台框架对比**
- 讲授：Flutter（Dart/Widget/状态管理）、React Native（JSX/原生桥接）
- 讲授：Uniapp（Vue/多端编译/条件编译）、MAUI（C#/跨平台 UI/平台特性）
- 对比：四大框架的技术路线、性能表现、生态成熟度
- 演示：TRAE 辅助生成跨平台基础代码

**第 3-4 学时：后端交互与硬件能力**
- 讲授：RESTful API 设计规范、JSON 数据解析、HTTP 请求库（Dio/Axios/HttpClient）
- 讲授：Token 认证机制（学生已学后端和软件工程，此处衔接）
- 讲授：移动设备传感器（GPS、加速度计、陀螺仪）接口调用
- 案例：物联网数据采集与展示场景

#### 💡 教学建议
- 后端交互部分可衔接学生已学的数据库原理和软件工程知识
- 传感器部分建议用真机演示（如手机加速度计数据实时展示）
- 四大框架不需要每个都深入讲，重点讲原理差异，实验中学生各选一个深入

#### 🏛️ 课程思政融入
- 通过"一次开发多端运行"案例，培养效率意识与系统思维

### 第四章：微信小程序开发（4 学时）

#### 🎯 教学目标
- 掌握微信小程序 MINA 框架、WXML/WXSS 语法、生命周期
- 掌握页面路由与数据绑定
- 了解小程序云开发基础（云函数、云数据库）
- 了解 Taro 框架多端适配（拓展）

#### 📝 教学设计

**第 1-2 学时：小程序基础**
- 讲授：MINA 框架结构、WXML/WXSS 语法
- 讲授：小程序生命周期（App/Page/Component）
- 演示：微信开发者工具使用、页面路由与数据绑定
- 实操引导：简单页面示例

**第 3-4 学时：云开发与拓展**
- 讲授：小程序云开发（云函数、云数据库）基础
- 演示：TRAE 辅助生成小程序页面代码
- 拓展介绍：Taro 框架实现多端小程序代码复用
- 案例：小程序在政务/公益领域的应用

#### 💡 教学建议
- 小程序开发门槛较低，学生容易上手，可多安排实操环节
- 云开发部分建议现场演示，让学生感受 Serverless 的便利
- Taro 作为拓展介绍即可，不要求深入掌握

#### 🏛️ 课程思政融入
- 结合小程序在政务/公益领域的应用，培养技术服务社会的价值观

### 第五章：鸿蒙多端应用开发（4 学时）【必须技术栈】

#### 🎯 教学目标
- **掌握** HarmonyOS NEXT 架构与生态
- **掌握** ArkUI/ArkTS 声明式 UI 语法与状态管理
- 理解多端部署与自适应布局策略
- 理解分布式能力原理（模拟器演示）
- 了解传感器数据采集与物联网应用场景

#### ⚠️ 教学重点
**本章为必须技术栈，所有同学必须掌握！**

#### 📝 教学设计

**第 1-2 学时：鸿蒙基础**
- 讲授：HarmonyOS NEXT 架构、与 Android/iOS 生态对比
- 讲授：ArkTS 声明式 UI 语法、状态管理
- 演示：DevEco Studio 开发环境
- 实操：创建第一个鸿蒙应用

**第 3-4 学时：多端适配与物联网**
- 讲授：手机/平板界面自适应布局策略（断点系统）
- 演示：分布式能力原理（通过模拟器演示跨设备数据同步）
- 讲授：传感器（光线传感器、陀螺仪等）数据采集
- 案例：设备间数据交互与物联网应用场景

#### 💡 教学建议
- **这是必须技术栈，要重点讲解，确保每位同学都能掌握**
- 分布式能力受实验室设备限制，以模拟器演示 + 理论讲解为主
- 传感器部分可结合真机演示（如光线传感器数据实时变化）
- **强调鸿蒙生态的国产化意义和发展前景，这是就业核心竞争力**

#### 🏛️ 课程思政融入
- 通过鸿蒙生态发展案例，培养民族品牌自信与技术创新精神
- 强调自主可控技术的重要性

### 第六章：综合开发实践（4 学时）

#### 🎯 教学目标
- 理解 MVP/MVVM 架构模式在移动应用中的实践
- 掌握多种数据存储方案
- 掌握性能优化基本方法
- 掌握 Git 版本控制与团队协作流程
- 学会利用 TRAE 进行代码重构与性能优化分析

#### 📝 教学设计

**第 1-2 学时：架构与数据**
- 讲授：MVP/MVVM 模式对比与实践
- 讲授：数据存储方案（SharedPreferences/SQLite/小程序缓存/鸿蒙数据管理）
- 讲授：性能优化（启动速度/UI 渲染/网络请求）

**第 3-4 学时：Git 协作与 AI 深度应用**
- 讲授：Git 分支管理、代码审查、协作开发流程
- 演示：Git 实操（clone/branch/commit/push/PR/merge）
- 演示：TRAE 进行代码重构与性能优化建议分析
- 总结：为实验六综合项目做准备

#### 💡 教学建议
- Git 部分建议现场实操演示，学生跟做
- 可结合实验六的项目需求，讲解如何组织团队协作
- AI 工具深度应用部分，展示 TRAE 在代码重构中的实际效果

#### 🏛️ 课程思政融入
- 通过项目迭代过程，培养精益求精的工匠精神与团队协作意识

## 🔬 实验教学指导

### 总体原则

#### AI 新范式模板化流程
所有实验均采用四阶段流程：**① 需求分析 → ② AI 辅助编码 → ③ 测试验证 → ④ 部署运维**。教师需在每个实验中引导学生按此流程组织开发，并在实验报告中体现各阶段过程。

#### 分组与技术栈分工
- 每组 6 人，按两个维度分组：
  - **项目业务组**：选择项目方向（智慧校园/智慧健康/智慧生活/智慧购物）
  - **开发技术组**：每人负责一个技术栈
- 实验二至实验五：每人独立完成对应技术栈任务，组内进行技术分享与对比分析
- 实验六：团队协作，每人负责一个技术栈端实现，使用 Git 协作

#### ⚠️ 鸿蒙必须技术栈
- **所有同学必须掌握鸿蒙开发技术**
- 实验五为必须参与的实验
- 期末大作业中必须体现鸿蒙技术栈成果

#### iOS 教学说明
实验室通常无 Mac 设备，iOS 部分处理方式：
- 理论课：教师使用 Mac 演示 SwiftUI 开发
- 实验课：选择 iOS 方向的学生可使用在线模拟器或观看教师录制的操作视频，重点撰写技术对比分析报告
- 评分时不因设备限制降低 iOS 方向学生成绩

### 实验一：开发环境搭建（2 学时）

#### 教学要点
- 提前准备各平台安装包/镜像，避免课堂下载耗时
- 每人负责一个平台环境，组内互助解决配置问题
- **重点配置鸿蒙开发环境（DevEco Studio）**

#### AI 新范式引导
- 需求分析：明确各平台版本要求
- AI 辅助：使用 TRAE 生成 Hello World 模板
- 测试：模拟器/真机验证
- 运维：记录配置文档

#### 常见问题预案
- Android Studio Gradle 下载慢：提前配置国内镜像
- Flutter SDK 配置：准备离线安装包
- DevEco Studio 版本兼容：确认 HarmonyOS NEXT 对应版本

### 实验二：原生应用开发（4 学时）

#### 教学要点
- Android 方向：Kotlin 登录页面（EditText 验证、Activity 跳转）
- iOS 方向：SwiftUI 登录页面（教师演示/模拟器）
- 组内对比分享：不同平台实现同一功能的差异

#### AI 新范式引导
- 需求分析：编写登录功能需求文档
- AI 辅助：TRAE 生成页面骨架代码
- 测试：输入验证边界条件测试
- 运维：打包调试版本，记录平台差异

#### 评分关注点
- 代码规范性、功能完整性
- 需求文档质量
- 组内技术对比分析的深度

### 实验三：跨平台应用开发（4 学时）

#### 教学要点
- 业务数据列表页：RESTful API 请求 + JSON 解析 + 下拉刷新
- 组内成员分别使用 Flutter/React Native/Uniapp/MAUI 实现
- 扩展：集成设备传感器（GPS/加速度计）

#### AI 新范式引导
- 需求分析：定义数据结构、API 接口规范、传感器需求
- AI 辅助：TRAE 生成网络请求与传感器调用代码
- 测试：API 异常响应容错、传感器数据准确性
- 运维：多框架性能对比报告

#### 后端 API 准备
- 建议教师提前准备统一的后端 API 服务，或指定公开 API（如 JSONPlaceholder）
- 提供 API 文档模板，统一 JSON 数据格式

### 实验四：微信小程序开发（4 学时）

#### 教学要点
- 通知小程序：列表 + 详情页路由 + 本地存储
- 重点体验 TRAE 在小程序代码生成中的作用

#### AI 新范式引导
- 需求分析：绘制页面流程图与数据模型
- AI 辅助：TRAE 生成页面代码与数据绑定逻辑
- 测试：微信开发者工具真机预览
- 运维：体验小程序发布流程（体验版）

### 实验五：鸿蒙多端应用开发（4 学时）【必须参与】

#### ⚠️ 教学重点
**这是必须技术栈实验，所有同学必须参与！**

#### 教学要点
- 天气应用：手机/平板自适应布局
- 分布式数据同步：模拟器演示
- 扩展：光线传感器、陀螺仪数据采集

#### AI 新范式引导
- 需求分析：功能需求 + 多端适配策略 + 传感器需求
- AI 辅助：TRAE 生成 ArkUI 布局与传感器代码
- 测试：不同设备模拟器验证 UI 适配
- 运维：多端适配与硬件集成分析报告

#### 设备限制处理
- 分布式能力以模拟器演示为主，不要求多设备实物组网
- 传感器功能在模拟器上受限时，可使用模拟数据

### 实验六：跨平台综合项目实战（6 学时）

#### 教学要点
- 团队项目：每人一个技术栈，Git 协作
- **建议选题涉及传感器/硬件交互（运动健康、智能家居等）**
- **必须包含鸿蒙技术栈成果**
- 使用 TRAE 辅助全流程开发

#### AI 新范式引导
- 需求分析：团队协作完成需求文档与技术选型
- AI 辅助：各成员使用 TRAE 辅助开发
- 测试：功能测试 + 跨端兼容性测试
- 运维：多端打包部署 + 技术选型对比报告

#### 教师角色
- 项目启动时审核各组需求文档与技术选型方案（**确保鸿蒙技术栈有人负责**）
- 开发过程中巡回指导，关注 Git 协作规范
- 项目结束时组织组内演示与答辩

## 📊 考核评价指导

### 成绩构成

| 考核项 | 比例 | 构成 |
|--------|:----:|------|
| 平时成绩 | 20% | 作业（50%）+ 课堂表现（20%）+ 阶段测验（30%） |
| 实验成绩 | 30% | 实验完成 + 技术栈深度 + 组内分享 + 实验报告 |
| 期末考查 | 50% | 大作业：代码 + 测试 + 文档 |

### ⚠️ 鸿蒙技术栈考核要求

**平时成绩**：阶段测验中包含鸿蒙技术栈内容（占比 15%）

**实验成绩**：
- 实验五（鸿蒙实验）为必做，占实验成绩的 25%
- 未完成鸿蒙实验者实验成绩最高记为 60 分

**期末大作业**：
- 必须包含鸿蒙技术栈成果
- 鸿蒙成果占期末成绩的 15%

### 评分标准要点

**平时成绩评分**：
- 作业：正确性、完成度、规范性
- 课堂表现：讨论参与度、互动质量
- 阶段测验：知识掌握程度（**含鸿蒙知识点**）

**实验成绩评分**：
- 注意区分四档（优秀/良好/合格/不合格）的描述差异
- 重点关注：AI 新范式四阶段过程记录、个人技术栈掌握深度、组内分享质量
- **鸿蒙实验必须完成，否则不予优秀**
- iOS 方向学生不因设备限制扣分，可通过技术分析报告补充

**期末大作业评分**：
- 目标 1（15%）：项目架构设计与技术选型分析
- 目标 2（25%）：RESTful API 对接、数据存储方案
- 目标 3（30%）：**多端适配方案对比（重点鸿蒙）**与选型论证
- 目标 4（30%）：AI 工具应用、代码重构、性能优化、Git 协作

### 课程目标达成度计算
各课程目标达成度 = (平时成绩 × 20% + 实验成绩 × 30% + 期末成绩 × 50%) / 该目标满分

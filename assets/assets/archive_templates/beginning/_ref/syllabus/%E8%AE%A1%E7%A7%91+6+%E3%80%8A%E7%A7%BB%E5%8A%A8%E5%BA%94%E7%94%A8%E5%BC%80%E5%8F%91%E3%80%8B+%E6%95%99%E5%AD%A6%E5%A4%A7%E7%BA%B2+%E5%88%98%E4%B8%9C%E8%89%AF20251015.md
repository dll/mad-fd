一、课程简介

移动应用开发是计算机科学与技术的一门专业限选课程，是一门应用型和实践性很强的课程。本课程的主要内容包括：移动应用开发技术体系（原生/混合/跨平台）、主流开发平台对比、Android/iOS
原生开发基础、混合开发框架（Flutter/React
Native、Cordova、Uniapp、Xamarin）、小程序开发（微信/支付宝）、华为多端应用技术、AI
编程工具在移动开发中的应用、UI 设计、数据存储、组件通信与跨平台适配等。本课程共 48
个学时，其中理论教学为 24 学时、实验教学为 24
学时。本课程的考评方式为考查，包括平时考核（20%）、实验考核（30%）和期终考核（50%）。

AI
编程工具的使用，具备跨平台应用系统分析、设计和开发能力，培养学生具备科学思维、创新意识和良好的职业道德，为从事移动开发工作及毕业设计奠定基础。

三、课程内容及要求

第一章 移动应用开发技术体系

教学内容：

1.  移动应用分类：原生应用、混合应用、小程序、多端应用的技术特征

2.  主流开发平台对比：Android/iOS 生态、微信/支付宝小程序平台、华为鸿蒙多端框架

3.  开发工具链：Android Studio、Xcode、HBuilderX、DevEco Studio、Visual Studio（Xamarin 开发）等

4.  AI 编程工具介绍：GitHub Copilot、TRAE等在移动开发中的辅助作用

5.  技术选型方法论：基于项目需求（性能/成本/跨平台性）的决策模型

6.  混合开发框架技术路线对比：原生代码共享（Flutter/Xamarin）与 Web 技术封装（Cordova/Uniapp）的差异

学生学习预期成果：能够辨析不同开发模式的技术差异，掌握主流工具链使用，了解 Xamarin 框架和 AI
编程工具的基本情况，为课程目标 1 提供支撑。

教学重点：技术体系分类及适用场景、开发工具链实操、Xamarin 框架简介。

教学难点：技术选型决策逻辑。

课程思政元素：通过国内外移动生态竞争案例，培养科技自立自强意识与技术判断力品格。

2.  原生开发基础

教学内容：

1.  Android 开发：Activity 生命周期、UI 控件基础（TextView/Button）

2.  iOS 开发入门：ViewController 架构、SwiftUI 基础组件

3.  原生应用调试：Logcat/Xcode 调试工具使用

学生学习预期成果：掌握原生开发核心概念，能够实现简单页面交互，支撑课程目标 1。

教学重点：Android/iOS 核心组件工作原理。

教学难点：原生平台特性差异。

课程思政元素：从系统底层逻辑培养严谨的工程思维与问题溯源能力。

3.  混合开发技术

教学内容：

1.  Flutter 框架：Dart 语法、Widget 组件、状态管理

2.  React Native：JSX 语法、原生模块桥接

3.  Cordova 框架：WebView 原理、插件机制、原生 API 调用

4.  Uniapp 框架：Vue 语法基础、多端编译原理、条件编译

5.  Xamarin 框架：

    Xamarin.Forms 跨平台 UI 组件

    C# 与原生 API 交互

    Xamarin.Android 和 Xamarin.iOS 平台特性

6.  混合应用打包：Android/iOS 发布流程

7.  AI 编程工具辅助开发：利用 GitHub Copilot 生成基础代码、调试建议

学生学习预期成果：使用混合框架开发跨平台页面，理解原生与 H5 交互原理，学会借助 AI
工具提高开发效率，支撑课程目标2。

教学重点：Flutter 组件布局与状态管理、Cordova插件机制、Uniapp多端适配。

教学难点：原生模块调用机制、不同框架技术路线的适用场景区分。

课程思政元素：通过 "一次开发多端运行" 案例，培养效率意识与系统思维。

4.  小程序开发

教学内容：

1.  微信小程序：MINA 框架、WXML/WXSS 语法、生命周期

2.  支付宝小程序：组件差异与生态能力（支付接口）

3.  跨平台适配：Taro 框架实现多端小程序代码复用

4.  AI 工具辅助小程序开发：使用 TRAE生成小程序页面代码

学生学习预期成果：独立开发具备基础功能的小程序，掌握跨平台适配技巧，能运用 AI
工具辅助开发，支撑课程目标 2。

教学重点：微信小程序页面路由与数据绑定。

教学难点：多平台小程序兼容性处理。

课程思政元素：结合小程序在政务/公益领域的应用，培养技术服务社会的价值观。

5.  华为多端应用开发

教学内容：

1.  鸿蒙生态：分布式任务调度、跨设备 UI 适配

2.  ArkUI 框架：声明式 UI 语法、状态管理

3.  多端部署：手机/平板/智能手表的界面适配策略

学生学习预期成果：使用 DevEco Studio 开发多端应用，实现设备间数据同步，支撑课程目标 3。

教学重点：分布式能力调用、跨设备 UI 适配。

教学难点：多设备协同逻辑设计。

课程思政元素：通过鸿蒙生态发展案例，培养民族品牌自信与技术创新精神。

6.  综合开发实践

教学内容：

1.  项目架构设计：MVP 模式在混合应用中的实践，Xamarin 项目结构设计

2.  数据存储方案：SharedPreferences、小程序本地缓存、鸿蒙分布式数据库、Xamarin 中的数据存储

3.  性能优化：启动速度优化、UI 渲染效率提升

4.  AI 工具深度应用：利用 AI 工具进行代码重构、性能优化建议分析

学生学习预期成果：整合多技术栈（包括 Xamarin）完成综合项目，能借助 AI
工具进行优化，具备系统优化能力，支撑课程目标 4。

教学重点：多技术栈集成方案设计、Xamarin 项目实战。

教学难点：跨技术栈调试与性能调优。

课程思政元素：通过项目迭代过程，培养精益求精的工匠精神与团队协作意识。

六、课程成绩评定

（一）考核方式及具体要求

最终成绩由平时成绩、实验成绩和期末考查成绩三部分组成，各部分所占比例如下：

平时成绩（占20%）：通过平时作业考核（50%）、课堂表现（20%）、阶段测验（30%）计算平时成绩。其中，平时作业考核：教师评判作业并根据作业内容的正确性、完成认真度及规范性给出评定成绩；课堂表现：教师根据学生课堂讨论、互动给出评定成绩；阶段测验：根据测试结果评定成绩。

实验成绩（占30%）：通过实验考核计算实验成绩。其中，实验考核根据实验项目完成情况以及实验报告的撰写质量给出评定成绩。

期末考查成绩（占50%）：重点考查学生应用移动应用开发技术解决问题的能力。期末考查采用大作业的形式，完成系统的代码编写、测试以及文档的撰写。

# 0105 移动应用开发能力培养图谱
*基于刘东良教师教学大纲制定*

## 能力培养体系概述
本图谱基于移动应用开发课程的4个课程目标，构建全方位的能力培养体系，涵盖技术应用能力、工程实践能力、创新设计能力和综合素质能力，对应支撑相关毕业要求的达成。

## 1. 技术体系掌握能力（对应课程目标1）

### 1.1 移动应用技术分类认知能力

- **原生应用特征识别**
  - 理解原生应用的技术特征：使用平台特定SDK，直接调用系统API
  - 掌握原生应用的性能优势：最优的运行性能和用户体验
  - 认识原生应用的开发成本：需要维护多套代码，开发成本较高
  - 明确原生应用的适用场景：游戏、高性能计算、系统级应用

- **混合应用架构理解**
  - 掌握混合应用技术特征：Web技术与原生容器结合
  - 理解混合应用开发效率优势：一套代码多平台运行
  - 认识混合应用性能表现：介于原生和Web应用之间
  - 明确混合应用适用场景：企业应用、内容展示类应用

- **小程序生态认知**
  - 理解小程序技术特征：运行在超级App内的轻量级应用
  - 掌握小程序生态依赖：依托微信、支付宝等平台生态
  - 认识小程序用户体验：免安装、即用即走
  - 明确小程序适用场景：轻量级服务、社交电商

- **多端应用概念掌握**
  - 理解多端应用技术特征：跨设备、跨平台的统一应用
  - 掌握分布式特性：设备间协同与数据同步
  - 认识华为鸿蒙特色：ArkTS语言、分布式软总线
  - 明确多端应用适用场景：IoT设备、智能家居、车联网

### 1.2 主流开发平台对比能力
- **Android/iOS生态分析**
  - Android：开放生态、市场占有率高、设备碎片化特点
  - iOS：封闭生态、用户价值高、开发规范严格特点
  - 技术选型考虑：目标用户群体、开发成本、上架要求

- **小程序平台差异识别**
  - 微信小程序：社交生态、用户基数大、API丰富
  - 支付宝小程序：支付生态、商业场景强、金融属性
  - 平台差异：开发规范、审核机制、生态能力

- **华为鸿蒙生态理解**
  - 分布式架构：跨设备协同、统一开发语言
  - 技术优势：原生性能、多端一致性
  - 生态发展：设备覆盖、开发者支持

### 1.3 开发工具链掌握能力
- **原生开发工具熟练使用**
  - Android Studio：Android官方IDE、完整工具链
  - Xcode：iOS官方IDE、Interface Builder、调试工具

- **跨平台开发工具应用**
  - HBuilderX：Uniapp官方IDE、多端调试
  - DevEco Studio：华为鸿蒙开发IDE、分布式调试
  - Visual Studio：Xamarin开发环境、C#跨平台

### 1.4 技术选型决策能力
- **性能需求分析**
  - 计算密集型：选择原生开发
  - UI密集型：考虑跨平台方案
  - 轻量级应用：小程序优先

- **成本效益分析**
  - 开发成本：人力、时间、维护
  - 平台覆盖：目标用户分布
  - 技术债务：长期维护成本

- **AI编程工具应用**
  - GitHub Copilot：代码自动补全、函数生成
  - CodeGeeX：中文友好的AI编程助手
  - AI工具在移动开发中的辅助作用和代码质量考虑

### 1.5 混合开发框架技术路线理解
- **原生代码共享型框架**
  - Flutter：Dart语言、Skia渲染引擎，接近原生性能
  - Xamarin：C#语言、.NET生态，学习成本较高

- **Web技术封装型框架**
  - Cordova/PhoneGap：HTML5+WebView封装
  - Uniapp：Vue语法、多端编译，Web技术栈

## 2. 跨平台开发能力（对应课程目标2）

### 2.1 Flutter框架开发能力
- **Dart语法基础掌握**
  - 变量声明：var、final、const的使用
  - 函数定义：参数传递、返回值处理
  - 类与对象：构造函数、继承关系
  - 异步编程：Future、async/await机制

- **Widget组件系统应用**
  - StatelessWidget：无状态组件开发
  - StatefulWidget：有状态组件开发
  - 常用Widget：Container、Row、Column、Stack使用
  - 自定义Widget：组件封装与复用

- **状态管理实现**
  - setState()：局部状态管理
  - Provider：全局状态管理
  - 数据流：父子组件通信
  - 生命周期：initState()、dispose()

- **网络数据处理**
  - HTTP请求：网络数据获取
  - JSON解析：数据格式转换
  - 异步处理：Future和Stream
  - 下拉刷新：用户交互优化

### 2.2 Uniapp多端开发能力
- **Vue语法基础**
  - 模板语法：数据绑定、指令使用
  - 组件系统：props、events、slots
  - 生命周期：created、mounted、destroyed
  - 路由管理：页面跳转、参数传递

- **多端编译原理**
  - 编译时适配：语法转换、API适配
  - 运行时适配：平台检测、功能降级
  - 条件编译：#ifdef、#endif使用
  - 平台差异处理：样式、组件、API

- **待办事项应用开发**
  - 数据绑定：Vue数据驱动界面
  - 列表渲染：v-for指令使用
  - 事件处理：用户交互响应
  - 本地存储：数据持久化

### 2.3 小程序开发能力
- **微信小程序MINA框架**
  - 双线程模型：逻辑层与渲染层分离
  - WXML模板语言：数据绑定、列表渲染、条件渲染
  - WXSS样式语言：选择器、单位、样式导入
  - JavaScript逻辑层：页面逻辑、数据处理

- **生命周期管理**
  - App生命周期：onLaunch、onShow、onHide
  - Page生命周期：onLoad、onShow、onReady、onHide、onUnload
  - Component生命周期：created、attached、ready、detached
  - 页面路由：wx.navigateTo、wx.redirectTo、wx.switchTab

- **校园通知小程序开发**
  - 列表页面：数据展示、交互设计
  - 详情页面：参数传递、内容展示
  - 路由跳转：页面间导航
  - 本地存储：wx.setStorageSync、wx.getStorageSync

### 2.4 支付宝小程序生态能力
- **组件差异分析**
  - 基础组件：view、text、image对比
  - 表单组件：input、button、picker差异
  - 导航组件：navigation-bar、tab-bar
  - API差异：网络请求、文件操作、设备信息

- **生态能力集成**
  - 支付接口：my.tradePay支付能力
  - 芝麻信用：信用分查询、授权
  - 生活服务：地理位置、联系人、日历
  - 营销工具：卡券、红包、积分

### 2.5 AI工具辅助开发能力
- **CodeGeeX应用**
  - 小程序页面代码生成：模板代码、样式代码
  - 组件逻辑实现：数据处理、事件响应
  - API调用代码：网络请求、本地存储
  - 调试优化建议：性能优化、错误修复

- **开发效率提升**
  - 代码自动补全：语法提示、API提示
  - 代码规范检查：ESLint规则、代码格式化
  - 重构建议：代码优化、结构调整
  - 文档生成：注释完善、API文档

  - 能力建设：有针对性地提升核心竞争力
  - 价值实现：在项目中体现个人技术价值
  - 影响力建设：通过技术贡献建立专业影响力

## 6. 能力培养评价体系

### 6.1 基于课程目标的能力评价
- **课程目标1评价（技术体系掌握）**
  - 评价标准：能熟练掌握移动应用开发技术体系分类和特点
  - 评价方法：理论考试、技术方案对比分析、工具使用测试
  - 达成指标：技术选型决策合理，开发工具使用熟练

- **课程目标2评价（跨平台开发）**
  - 评价标准：能独立完成跨平台应用开发项目
  - 评价方法：实验项目评估、代码质量审查、功能实现测试
  - 达成指标：Flutter、Uniapp、小程序项目成功运行

- **课程目标3评价（多端适配）**
  - 评价标准：能实现华为多端应用的跨设备适配
  - 评价方法：鸿蒙应用开发、多设备测试、数据同步验证
  - 达成指标：天气应用在不同设备上正常运行

- **课程目标4评价（工程实践）**
  - 评价标准：能遵循软件工程规范完成项目开发
  - 评价方法：综合项目评估、文档质量审查、团队协作评价
  - 达成指标：校园服务应用开发完整，技术选型报告合理

### 6.2 分层次能力评价标准
- **优秀（90-100分）**
  - 全面掌握移动应用开发技术体系
  - 能独立完成复杂跨平台项目开发
  - 具有创新能力和优秀的工程实践能力
  - 团队协作优秀，职业素养高

- **良好（80-89分）**
  - 较好掌握移动应用开发技术
  - 能完成一般跨平台项目开发
  - 具有较好的工程实践能力
  - 团队协作良好，职业素养较好

- **合格（60-79分）**
  - 基本掌握移动应用开发技术
  - 能在指导下完成项目开发
  - 具备基本的工程实践能力
  - 团队协作基本合格，职业素养一般

- **不合格（<60分）**
  - 技术掌握不足，难以独立完成项目
  - 工程实践能力弱
  - 团队协作能力不足，职业素养欠缺

### 6.3 持续改进机制
- **能力诊断**：定期评估学生能力发展状况
- **个性化指导**：针对不同学生制定培养方案
- **实践强化**：通过项目实战提升能力水平
- **反馈优化**：根据评价结果调整培养策略

## 7. 能力培养实施路径

### 7.1 分阶段能力培养计划
- **第1-2周**：技术体系认知阶段
  - 重点：移动应用分类理解、开发平台对比
  - 方法：理论学习、案例分析、环境搭建
  - 目标：建立完整的技术体系认知框架

- **第3-5周**：跨平台开发基础阶段
  - 重点：Flutter、Uniapp、小程序基础开发
  - 方法：实验实践、代码编写、功能实现
  - 目标：掌握主流跨平台开发技术

- **第6-7周**：多端适配进阶阶段
  - 重点：鸿蒙应用开发、设备适配
  - 方法：项目实践、真机测试、性能优化
  - 目标：具备多端应用开发和适配能力

- **第8周**：工程实践综合阶段
  - 重点：项目架构设计、团队协作开发
  - 方法：综合项目、团队合作、质量控制
  - 目标：形成完整的工程实践能力

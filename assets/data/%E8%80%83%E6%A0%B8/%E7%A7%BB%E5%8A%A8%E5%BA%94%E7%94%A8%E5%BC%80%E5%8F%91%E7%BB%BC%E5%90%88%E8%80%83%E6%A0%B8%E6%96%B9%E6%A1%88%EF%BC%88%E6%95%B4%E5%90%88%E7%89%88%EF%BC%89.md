# 移动应用开发综合考核方案（整合版）

## 前言

本次考核聚焦多平台移动应用开发技术的综合应用，强调团队协作与跨框架整合能力。共设 4 道题目，每组 6 名学生分工负责不同技术栈开发，最终形成多端协同的完整系统。考核涵盖原生开发、跨平台框架、混合应用及微软生态技术，注重技术特色与跨端交互能力的评估。

## 题目一：校园生活服务平台开发与整合

### 一、业务系统描述

开发 "校园生活服务平台"，核心功能包括：

- 校园资讯模块（通知公告、校园动态、院系新闻）
- 服务中心（失物招领、校园报修、场馆预约、请假申请）
- 生活工具（校园地图导航、校车时刻表、作息日历、教室查询）
- 个人中心（我的预约、消息通知、个人资料、设置）

### 二、组内分工（6 人 / 组）

1. **成员 1**：Android 原生应用开发（Kotlin）
2. **成员 2**：iOS 原生应用开发（Swift）
3. **成员 3**：Flutter 跨平台应用开发
4. **成员 4**：Cordova 混合应用开发（兼容 Android/iOS）
5. **成员 5**：Xamarin.Forms 开发（微软框架，支持多平台）
6. **成员 6**：微信小程序开发

### 三、技术参考方案

#### 基础要求

各平台实现核心功能，遵循统一设计规范（含颜色体系、组件样式、交互逻辑），保证基础 UI 一致性。

#### 平台特色技术

- **Android 端**：
  集成华为 / 小米推送服务，实现后台消息唤醒；使用 Jetpack 组件（ViewModel、Room）优化数据管理；冷启动时间＜3 秒。
- **iOS 端**：
  适配暗黑模式与动态岛（iPhone 14+）快捷操作；使用 SwiftUI 构建响应式界面；集成 iOS 共享菜单实现资讯快速分享。
- **Flutter 端**：
  实现与原生模块通信（如调用 Android/iOS 地图 SDK）；使用 Bloc 模式管理全局状态；优化列表滑动性能（≥60fps）。
- **Cordova 端**：
  开发 2 个自定义插件（校园地图定位插件、设备信息获取插件）；使用 Cordova CLI 实现自动化构建；通过`cordova-plugin-whitelist`解决跨域问题。
- **Xamarin 端**：
  采用 MVVM 模式架构；集成 Xamarin.Essentials 访问设备传感器（如定位）；实现与.NET 后端（[ASP.NET](https://asp.net/) Core）的数据交互。
- **微信小程序**：
  接入微信授权登录；实现场馆预约的微信支付模拟；使用小程序云开发存储用户发布的失物招领信息。

### 四、整合要求

1. 搭建统一后端 API（推荐[ASP.NET](https://asp.net/) Core 或 Node.js），实现用户数据（登录状态、预约记录等）多端同步。
2. 开发 "平台入口" 页面，提供各端应用的下载 / 访问链接及功能对比说明。
3. 实现跨平台数据互通：用户在任意端发布的信息（如失物招领），其他端实时可见。
4. 跨框架交互：Cordova 应用通过 JavaScript 桥接调用 Xamarin 的文件处理模块（如导出报修记录为 PDF）。

## 题目二：健康运动记录平台开发与整合

### 一、业务系统描述

开发 "健康运动记录平台"，核心功能包括：

- 运动记录（跑步、骑行、步行的轨迹、时长、配速等数据统计）
- 目标管理（每日 / 每周运动目标设置、达成进度统计、勋章体系）
- 数据中心（运动历史记录、卡路里消耗趋势图表、月度报告）
- 社交互动（运动动态分享、好友排行榜、运动群组挑战）
- 个人中心（运动偏好设置、数据导出、第三方健康平台绑定）

### 二、组内分工（6 人 / 组）

1. **成员 1**：Android 原生应用开发（集成运动传感器）
2. **成员 2**：iOS 原生应用开发（集成 HealthKit）
3. **成员 3**：Flutter 跨平台应用开发
4. **成员 4**：Cordova 混合应用开发
5. **成员 5**：MAUI 开发（微软框架，替代 Xamarin）
6. **成员 6**：Uniapp 开发（兼容 H5 + 支付宝小程序）

### 三、技术参考方案

#### 基础要求

各平台支持运动数据采集（自动 + 手动录入）、数据展示与基础社交功能，支持离线记录与联网同步。

#### 平台特色技术

- **Android 端**：
  调用计步传感器与 GPS 实现自动轨迹记录；集成 Google Fit 同步数据；使用 WorkManager 实现后台持续计步。
- **iOS 端**：
  接入 HealthKit 获取系统运动数据；实现 Siri 快捷指令（如 "开始跑步"）；使用 Core Location 优化轨迹精度。
- **Flutter 端**：
  使用`charts_flutter`绘制运动趋势图表；通过`geolocator`插件获取定位；使用 Hive 数据库实现离线数据缓存。
- **Cordova 端**：
  使用`cordova-plugin-device-motion`获取加速度数据；开发后台持续计步插件（解决 WebView 休眠问题）；集成`cordova-plugin-health`同步系统健康数据。
- **MAUI 端**：
  实现 Windows + 移动设备跨平台部署；使用.NET MAUI Charts 绘制数据图表；集成系统健康服务（Android Health Connect、iOS HealthKit）；通过 Blazor Hybrid 混合 Web 与原生控件。
- **Uniapp 端**：
  通过条件编译适配 H5 与支付宝小程序；使用支付宝运动 API 同步步数；通过云函数实现排行榜实时计算。

### 四、整合要求

1. 实现运动数据多端同步（支持离线记录，联网后自动增量同步），设计数据冲突解决机制（如同一时段多端数据合并规则）。
2. Cordova 应用与 MAUI 应用共享.NET 后端的运动数据分析服务（如卡路里计算、运动强度评估）。
3. 开发统一数据看板，对比各平台数据采集精度（如 GPS 轨迹偏差、计步误差）。
4. 实现跨平台社交互动：Uniapp 用户可查看 Flutter 端用户的运动动态并点赞。

## 题目三：在线学习辅助平台开发与整合

### 一、业务系统描述

开发 "在线学习辅助平台"，核心功能包括：

- 课程中心（课程列表、详情、章节学习、教师信息）
- 学习工具（笔记记录、重点标记、学习时长统计、番茄钟）
- 练习模块（章节测验、错题本、模拟考试、答案解析）
- 资源库（课件下载、拓展阅读、视频教程、代码示例）
- 个人中心（学习进度、证书查询、学习报告、同学圈）

### 二、组内分工（6 人 / 组）

1. **成员 1**：Android 原生应用开发
2. **成员 2**：iOS 原生应用开发
3. **成员 3**：Flutter 跨平台应用开发
4. **成员 4**：Cordova 混合应用开发
5. **成员 5**：Xamarin.Forms 开发
6. **成员 6**：微信小程序开发

### 三、技术参考方案

#### 基础要求

各平台实现课程浏览、学习进度记录与基础笔记功能，保证学习数据互通。

#### 平台特色技术

- **Android 端**：
  实现视频离线下载与加密播放（AES 加密）；支持悬浮窗学习（Picture-in-Picture）；使用 Room 缓存课程数据。
- **iOS 端**：
  支持画中画播放与 Split View 分屏学习；接入 ARKit 实现 3D 课件展示；使用 Core Data 管理离线资源。
- **Flutter 端**：
  使用`webview_flutter`集成在线编程练习环境；通过`sqflite`实现笔记本地存储；使用`video_player`优化视频播放体验。
- **Cordova 端**：
  使用`cordova-plugin-file`实现课程资源本地存储；开发自定义加密插件保护付费课程；通过预加载与资源压缩优化 WebView 加载速度。
- **Xamarin 端**：
  集成 Microsoft Graph API 访问 OneDrive 学习资料；实现 Windows 平板与手机的 Continuum 无缝切换；使用 Prism 框架实现 MVVM 架构。
- **微信小程序**：
  实现课程分享与拼团功能；集成微信支付模拟购课；使用小程序云开发存储用户笔记与学习记录。

### 四、整合要求

1. 实现学习进度多端同步（如 Android 端学习的章节，iOS 端自动标记为 "已学"）。
2. 开发统一课程管理后台（推荐[ASP.NET](https://asp.net/) Core MVC），支持各平台课程数据更新。
3. 跨框架交互：Cordova 应用通过 JavaScript 桥接调用 Xamarin 的 PDF 批注功能（如课件重点标记）。
4. 设计统一的视频播放进度同步机制（精确到秒级）。

## 题目四：二手物品交易平台开发与整合

### 一、业务系统描述

开发 "二手物品交易平台"，核心功能包括：

- 商品模块（分类浏览、详情展示、搜索筛选、价格趋势）
- 交易功能（发布商品、出价 / 议价、订单管理、支付流程）
- 沟通工具（在线聊天、消息通知、交易提醒）
- 信用体系（评价管理、交易记录、信用分计算）
- 个人中心（我的发布、收藏、交易统计、地址管理）

### 二、组内分工（6 人 / 组）

1. **成员 1**：Android 原生应用开发
2. **成员 2**：iOS 原生应用开发
3. **成员 3**：Flutter 跨平台应用开发
4. **成员 4**：Cordova 混合应用开发
5. **成员 5**：MAUI 开发
6. **成员 6**：Uniapp 开发（兼容 H5 + 支付宝小程序）

### 三、技术参考方案

#### 基础要求

各平台实现商品发布、浏览、交易核心流程，支持图片展示与消息交互。

#### 平台特色技术

- **Android 端**：
  实现商品图片压缩与水印（防盗用）；集成高德 / 百度地图显示交易地点；使用 Jetpack Compose 构建商品详情页。
- **iOS 端**：
  支持相册多选与编辑（裁剪、滤镜）；使用 Core ML 实现商品自动分类（如 "电子产品"" 书籍 "）；支持 Face ID 登录验证。
- **Flutter 端**：
  实现商品详情页缓存（减少重复请求）；使用`cached_network_image`优化图片加载；通过`flutter_screenutil`适配多屏幕。
- **Cordova 端**：
  开发图片处理自定义插件（裁剪、水印、压缩）；使用`cordova-plugin-inappbrowser`实现支付流程；通过`cordova-plugin-ionic-webview`优化 WebView 性能。
- **MAUI 端**：
  实现多设备适配（手机 / 平板 / PC）；使用 Blazor Hybrid 集成 Web 视图与原生控件；开发商家管理后台功能（商品审核、订单处理）。
- **Uniapp 端**：
  通过 AI 插件实现商品智能定价建议（基于历史交易数据）；适配支付宝支付；使用`uView UI`组件库优化交互体验。

### 四、整合要求

1. 实现商品与订单数据多端实时同步（基于 WebSocket）。
2. MAUI 开发的管理后台能直接管理 Cordova 应用提交的商品数据（审核、下架等）。
3. 设计统一的图片存储与加载策略（缩略图 + 原图分离，CDN 加速）。
4. 实现跨平台商品分享：Uniapp 商品可生成带参数二维码，Flutter 端扫码直接打开对应商品页。

## 提交成果与评分标准

### 一、提交成果

1. 各平台源代码（含配置文件、插件代码）及可运行安装包（小程序提供体验码）
2. 后端 API 文档（含接口定义、数据格式）与数据库设计
3. 整合测试报告（含跨平台功能测试用例、数据同步验证结果）
4. 演示视频（8-10 分钟，展示各平台功能及跨框架交互效果）
5. 技术文档（含架构设计、团队分工记录、技术难点解决方案）

### 二、评分标准（100 分）

| 评分项         | 分值  | 评价标准                                                     |
| -------------- | ----- | ------------------------------------------------------------ |
| 功能完整性     | 25 分 | 核心模块实现度，无关键功能缺失；边缘场景（如网络异常）处理合理性 |
| 平台技术特色   | 20 分 | 各平台特色功能实现质量（如 Cordova 插件完整性、MAUI 跨设备适配效果） |
| 跨框架整合效果 | 25 分 | 数据同步一致性（≤500ms 延迟）；跨框架交互流畅度；多端协同体验 |
| 性能与兼容性   | 15 分 | 各平台运行流畅（无明显卡顿）；适配主流设备版本；内存占用合理 |
| 文档与团队协作 | 15 分 | 文档完整性与规范性；Git 版本控制记录完整性；分工合理性与协作效率 |

## 考核实施说明

1. 考核周期为 3 周（15 个工作日），包含 1 次中期检查（第 7 个工作日）
2. 代码需托管在 GitHub/Gitee，每日提交代码（至少 1 次有效提交）
3. 个人得分 = 团队得分 × 个人贡献系数（根据负责模块质量、代码量、文档完整性评定）
4. 禁止抄袭，一经发现取消考核资格

本方案通过统一业务场景与差异化技术要求，既考察学生对各平台技术的掌握深度，又评估跨框架整合与团队协作能力，符合企业实际开发场景需求。

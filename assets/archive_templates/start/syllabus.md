  []三、课程内容及要求

  第一章 移动应用开发技术体系

  教学内容：

  [] 移动应用分类：原生应用、混合应用、小程序、多端应用的技术特征与适用场景。

  [] 主流开发平台对比： Android/iOS 生态、微信小程序平台、华为鸿蒙多端框架。

  . 开发工具链： Android Studio、Xcode、 HBuilderX、 DevEco Studio、Visual Studio（ MAUI）等。

  [] AI 编程工具： TRAE 等在移动开发中的辅助作用。

  . 技术选型方法论：基于项目需求（性能/成本/跨平台性/团队技术栈）的决策模型。

  . 跨平台技术路线对比：原生代码共享（Flutter/MAUI）、JS 桥接（React Native）与 Web
  技术封装（Uniapp）的差异。

  学生学习预期成果： 能够辨析不同开发模式的技术差异，掌握主流工具链使用，了解各跨平台框架和 AI
  编程工具的基本情况，为课程目标 1 提供支撑。

  教学重点： 技术体系分类及适用场景、开发工具链实操。

  教学难点： 技术选型决策逻辑。

  课程思政元素： 通过国内外移动生态竞争案例，培养科技自立自强意识与技术判断力品格。

  第二章 原生开发基础

  教学内容：

  [] Android 开发： Kotlin 语言基础、 Activity 生命周期、 UI 控件（TextView/Button/RecyclerView）、
  Logcat 调试。

  [] iOS 开发概述： ViewController 架构、 SwiftUI 基础组件、 Xcode 调试工具（理论讲授与演示为主）。

  [] 原生与跨平台对比：原生开发的性能优势与开发成本分析。

  学生学习预期成果： 掌握 Android 原生开发核心概念与 Kotlin 基础语法，能够实现简单页面交互；理解 iOS
  开发基本架构与 SwiftUI 组件模型，支撑课程目标 1。

  教学重点： Android Kotlin 开发核心组件、 Activity 生命周期。

  教学难点： 原生平台特性差异、 Kotlin 与 Java 的对比选择。

  课程思政元素： 从系统底层逻辑培养严谨的工程思维与问题溯源能力。

第三章 跨平台应用开发

  教学内容：

  . Flutter 框架： Dart 语法、Widget 组件、状态管理（Provider/Bloc）。

  [] React Native 框架：JSX 语法、原生模块桥接、组件生命周期。

  [] Uniapp 框架：Vue 语法基础、多端编译原理、条件编译。

  [] MAUI 框架： C# 跨平台 UI 组件、原生 API 交互、 MAUI.Android 和 MAUI.iOS 平台特性。

  . 后端交互： RESTful API 设计规范、JSON 数据解析、 HTTP 请求库（Dio/Axios/HttpClient）、Token
  认证机制。

  [] 移动硬件能力调用：设备传感器（GPS、加速度计、陀螺仪）接口、物联网数据采集与展示基础。

  [] AI 编程工具辅助开发：利用 TRAE 生成基础代码、调试建议。

学生学习预期成果： 使用跨平台框架开发应用页面，掌握 RESTful API 数据交互流程，理解原生与 H5
交互原理，学会借助 AI 工具提高开发效率，支撑课程目标 2。

  教学重点： Flutter 组件布局与状态管理、 Uniapp 多端适配、 RESTful API 交互实现。

  教学难点： 原生模块调用机制、不同框架技术路线的适用场景区分、 Token 认证流程。

  课程思政元素： 通过一次开发多端运行案例，培养效率意识与系统思维。

第四章 微信小程序开发

  教学内容：

  [] 微信小程序： MINA 框架、WXML/WXSS 语法、生命周期、页面路由与数据绑定。

  [] 小程序云开发：云函数、云数据库基础。

  [] 跨平台适配： Taro 框架实现多端小程序代码复用（拓展介绍）。

  [] AI 工具辅助小程序开发：使用 TRAE 生成小程序页面代码。

  学生学习预期成果： 独立开发具备基础功能的微信小程序，掌握小程序生命周期与本地存储，能运用 AI
  工具辅助开发，支撑课程目标 2。

  教学重点： 微信小程序页面路由、数据绑定与云开发基础。

  教学难点： 小程序性能优化与兼容性处理。

  课程思政元素： 结合小程序在政务/公益领域的应用，培养技术服务社会的价值观。

第五章 鸿蒙多端应用开发

  教学内容：

  [] 鸿蒙应用开发概述： HarmonyOS NEXT 架构、与 Android/iOS 生态对比。

  [] ArkUI 框架：ArkTS 声明式 UI 语法、状态管理。

  . 多端部署：手机/平板界面自适应布局策略。

  [] 分布式能力：跨设备数据同步原理（理论讲授，模拟器演示）。

  [] 移动硬件与物联网扩展：传感器（光线传感器、陀螺仪等）数据采集、设备间数据交互与物联网应用场景。

  学生学习预期成果： 使用 DevEco Studio 开发鸿蒙应用，实现多端 UI
  适配，理解分布式能力原理，支撑课程目标 3。

  教学重点： ArkUI 声明式语法、多端 UI 适配策略。

  教学难点： 分布式能力的设计理念与实现机制。

  课程思政元素： 通过鸿蒙生态发展案例，培养民族品牌自信与技术创新精神。

  第六章 综合开发实践

  教学内容：

  [] 项目架构设计： MVP/MVVM 模式在移动应用中的实践。

  [] 数据存储方案： SharedPreferences、SQLite、小程序本地缓存、鸿蒙数据管理。

  [] 性能优化：启动速度优化、 UI 渲染效率提升、网络请求优化。

  [] Git 版本控制与团队协作：分支管理、代码审查、协作开发流程。

  [] AI 工具深度应用：利用 AI 工具进行代码重构、性能优化建议分析。

  学生学习预期成果： 整合多技术栈完成综合项目，能借助 AI 工具与 Git
  进行团队协作开发与优化，具备系统优化能力，支撑课程目标 4。

  教学重点： 多技术栈集成方案设计、 Git 协作流程。

  教学难点： 跨技术栈调试与性能调优。

  课程思政元素： 通过项目迭代过程，培养精益求精的工匠精神与团队协作意识。

  测试验证：在模拟器/真机上验证项目运行；④ 部署运维：记录环境配置文档，总结常见问题与解决方案。

  学生学习预期成果：

  成功运行各平台实验项目，掌握环境配置流程与 AI 新范式开发流程，从而支撑课程目标 1 的达成。

  主要仪器设备： 微型计算机。

  实验项目二：原生应用开发

  实验内容：

  组内每人选择不同平台或技术方案实现同一功能，完成后进行组内技术对比分享。

  AI 新范式开发流程：

  ① 需求分析：编写登录功能需求文档（输入验证规则、页面跳转逻辑）；② AI 辅助编码：使用 TRAE
  生成登录页面骨架代码并手动完善业务逻辑；③ 测试验证：编写输入验证测试用例，验证边界条件；④
  部署运维：打包调试版本，记录平台差异对比报告。

  学生学习预期成果：

  掌握 Android Activity 跳转与数据传递，理解 iOS 开发基本架构，体验 AI
  新范式全流程开发，从而支撑课程目标 1 的达成。

  主要仪器设备： 微型计算机。

① 需求分析：定义列表页数据结构、 API 接口规范与传感器调用需求；② AI 辅助编码：使用 TRAE

  学生学习预期成果：

  适配效果与传感器数据准确性；④部署运维：编写多端适配与硬件集成技术分析报告。

  学生学习预期成果：

  掌握 ArkUI 声明式语法与多端 UI
  适配策略，理解分布式能力原理，了解移动设备传感器与物联网集成基础，从而支撑课程目标 3 的达成。

  主要仪器设备： 微型计算机。

  实验项目六：跨平台综合项目实战

  Native/Uniapp/MAUI/微信小程序），使用 Git 进行版本控制与协作，借助 AI 编程工具辅助代码生成与调试。

  []六、课程成绩评定

  （一）考核方式及具体要求

  最终成绩由平时成绩、实验成绩和期末考查成绩三部分组成，各部分所占比例如下：

  平时成绩（占 20%）：
  通过平时作业考核（50%）、课堂表现（20%）、阶段测验（30%）计算平时成绩。其中，平时作业考核：教师评判作业并根据作业内容的正确性、完成认真度及规范性给出评定成绩；课堂表现：教师根据学生课堂讨论、互动给出评定成绩；阶段测验：根据测试结果评定成绩。

  实验成绩（占 30%）：
  通过实验考核计算实验成绩。其中，实验考核根据实验项目完成情况、个人技术栈掌握深度、组内技术分享质量以及实验报告的撰写质量给出评定成绩。

  期末考查成绩（占 50%）：
  重点考查学生应用移动应用开发技术解决问题的能力。期末考查采用大作业的形式，完成系统的代码编写、测试以及文档的撰写。

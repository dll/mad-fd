## 三、技术架构里程碑

### 3.1 多智能体（24 个）

```
AgentRegistry (单例)
  ├── BaseAgent (抽象基类)
  │   ├── loadEffectivePersona — 优先 .md 后 fallback const
  │   ├── safeAiChatWithMeta — 自动写 agent_call_logs（含 chainId）
  │   ├── buildRagPrompt — 向量 RAG 优先 / TF-IDF 兜底
  │   └── safeAiChatWithTools — 工具调用循环
  ├── OrchestratorAgent — 多 Agent 串联（含 chainId 追溯）
  └── 24 专业 Agent：
       教学：tutor / mobile_expert / madkg / learning / path
       批阅：lab_grading / assessment_grading / works_grading
       辅助：quiz / lab / assessment / works / achievement / repo / graph
       安全：safety / ethics
       数字孪生：virtual_student / virtual_teacher
       生成：courseware / course_gen / doc_converter
       入口：voice / assistant
```

**Prompt 配置化**：所有 24 Agent persona 抽到 `assets/agent_prompts/{id}.md`，每份 50-130 行真定制内容；BaseAgent 启动加载，后续教学迭代只改 .md 不动代码。

### 3.2 RAG 检索增强

```
启动时 RagBootstrapService.ensureIndexed() 后台 unawaited 调用：
  ├── concepts → indexDocument → rag_embeddings
  ├── resources → indexDocument
  └── questions → indexDocument

BaseAgent.buildRagPrompt：
  if (rag_embeddings.count > 0):
    retrieveContextVector  ← embedding + 余弦 top-K
  else:
    retrieveContext  ← TF-IDF 兜底

EmbeddingService LRU 缓存 maxSize=64：
  按 (provider|model|text) 复合 key
```

### 3.3 多 Agent 协作（Orchestrator）

```
教师批阅页打开「增强批阅」Switch：
  OrchestratorAgent.runChain(chain=['safety','lab_grading','ethics'])
    ↓ 通过 Zone 注入 chainId
  BaseAgent 每步调用 → agent_call_logs.chain_id 自动写入

教师工作台 → AI 调用统计 Dashboard：
  Tab1: Agent 排行（COUNT/AVG/CHARS）
  Tab2: 调用链路（按 chainId 聚合）→ 点击展开 step 详情
```

### 3.4 数据同步（无服务器）

```
学生设备 ──[uploadStudentData]──→ Gitee 仓库 ──[downloadStudentData]──→ 教师设备
  ↑                                                                        ↓
  ←─────[downloadOwnData 拉教师批阅结果]──────────────────────────────────
```

**关键设计**：
- task_id 重映射：跨设备 ID 不同，按 title 自然键匹配
- 批改数据保护：已批改 lab_submissions 不被 import 覆盖
- 即时同步：unawaited(uploadStudentData) 不阻塞主流程

### 3.5 隐私合规

- **协议页**：登录页底部入口 → 用户协议 + 隐私声明双 Tab（flutter_markdown 渲染）
- **我的数据页**：22 表 Future.wait 并行 SELECT → JSON 导出
- **删除我的数据**：单事务批量 DELETE（保留教学档案：已核准 achievement_scores）
- **设计权衡**：本地清，远程联系管理员（Gitee 共享仓库的取舍）

## 七、关键架构决策记录

### 7.1 单仓共享 Gitee Token

**取舍**：所有学生 / 教师共用同一个 Gitee token（写在 GiteeCredentials.syncToken）。

**优**：无服务器，部署成本 0；学生秒级同步可见。

**劣**：恶意学生理论上可破坏其他人提交。

**对抗手段**：
- 隐私声明明确告知用户
- 教师批阅核准后写 grading_results 留审计
- 平台不收集敏感 PII

### 7.2 Prompt 配置化路径

**版本演进**：
- v0：const string 硬编码在 .dart 类
- v1：PromptLoader 框架（仅 3/24 真 .md，覆盖 12.5%）
- v2：13/24 = 54%
- v3：24/24 = 100% ✅

**未来**：教学一学期后形成 Prompt v2 实战版。

### 7.3 OrchestratorAgent chainId 串联

**问题**：3 步 Agent 调用是 3 条独立 agent_call_logs 记录，无法关联"这次实验批阅总耗时"。

**方案**：通过 `runZoned(zoneValues: {#agentChainId, #agentChainStep})` 注入；BaseAgent.safeAiChatWithMeta finally 块从 Zone.current 读取写入日志。

**收益**：listByChain / listRecentChains DAO 查询天然支持，Dashboard Tab2 直接复用。

### 7.4 part / part of 拆分模式

**约束**：Dart part 只能拆顶层 class，不能拆 State 内成员。

**适用**：经典 Tab 结构（works / classroom / git_repo）— 每个 _XxxTab 独立顶层私有 class，可平直抽到 part。

**不适用**：单 State 内嵌（courseware_workshop / graph_detail State 部分）— 需 helper class 重构，本期搁置。

### 7.5 多 Agent 协作的"成本 / 价值"权衡

**默认**：教师批阅走单 Agent（lab_grading），低成本。

**Switch 启用**：增强批阅时 safety→grading→ethics 三 Agent 链，约 3 倍 token，适合疑似 AI 代写 / 涉敏内容。

# Flutter 知识图谱学习系统测试用例文档

## 1. 文档信息

- **项目名称**：移动应用开发知识图谱学习系统
- **技术框架**：Flutter + Dart + SQLite
- **文档类型**：测试用例说明
- **适用范围**：图谱、资料、视频、测验、进度、收藏、登录等核心功能
- **版本**：v1.0

---

## 2. 测试目标

本测试用例文档用于说明 Flutter 知识图谱学习系统的主要测试点，覆盖以下目标：

1. 验证核心页面是否能够正常加载与显示
2. 验证知识图谱模块是否能够正常读取并展示数据
3. 验证节点交互、收藏、学习记录等关键行为是否符合预期
4. 验证资料、视频、测验、进度等模块是否能与图谱形成学习闭环
5. 验证数据模型转换、业务逻辑判断和本地数据读写是否正确
6. 为项目汇报、教学展示、测试报告编写提供依据

---

## 5. 测试策略

### 5.1 测试层次

1. **模型测试**  
   验证 `fromMap`、`toMap`、派生属性、角色判断、正确答案判断等逻辑

2. **界面测试**  
   验证页面关键元素是否正确显示，如标题、按钮、导航项、输入框等

3. **功能流程测试**  
   验证从首页进入图谱、图谱加载、节点交互、收藏、学习记录、测验等核心流程

4. **构建验证测试**  
   验证项目是否能够正常执行测试并输出 Android APK

### 5.2 通过标准

- 测试代码可正常执行
- 关键页面无明显崩溃
- 关键模型转换逻辑正确
- 主要业务路径符合预期
- APK 构建成功
